# WebApp: Финансы

**Теги:** #webapp #финансы #транзакции #дебиторка

## Суть

Финансовый модуль является «источником истины» для всей системы. Здесь фиксируются все движения денежных средств, обязательства и формируется управленческая отчетность.

## Страница `/finance` — структура вкладок

Пять вкладок-кнопок в верхней строке:

| ID         | Заголовок         | Подпись                   | Сумма на кнопке                 |
| ---------- | ----------------- | ------------------------- | ------------------------------- |
| `expenses` | Расходы           | структура и аналитика     | ✅ сумма за месяц               |
| `income`   | Доходы            | поступления за месяц      | ✅ сумма за месяц               |
| `recv`     | Дебиторка         | долги контрагентов        | ✅ общий долг                   |
| `loans`    | Кредиты и лизинг  | обязательства компании    | ✅ остаток                      |
| `payables` | Долги поставщикам | Дерябин, Новиков, Ромашин | ❌ не показывается (не влезает) |

## Панель «Расходы» (`expenses`)

- **Чипы:** Все расходы / 📌 Постоянные / ⚡ Переменные
- **Период:** День / Неделя / Месяц с навигацией ← →
- **Сводные карточки:** Постоянные / Переменные / Прибыль (за месяц) или Расходы (за период)
- **Двухколоночный макет:** слева — таймлайн транзакций, справа — структура
- **Структура расходов (правая колонка):** StackBar + LegendRow по группам + мини-карточки Постоянные%/Переменные%
- **Группировка:** `EXPENSE_GROUPS` — статические группы по ключевым словам в категории/описании (Зарплата, ГСМ, Запчасти, Ремонт, Кредиты/Лизинг, Налоги, Прочие)
- **Клик на LegendRow** → фильтр таймлайна по группе

## Панель «Доходы» (`income`)

Архитектурно повторяет панель «Расходы» (реализовано в сессии 13, 18.05.2026).

- **Чипы:** Все доходы / 🚛 Рейсы / 💳 Поступления
- **Период:** День / Неделя / Месяц с навигацией ← →
- **Сводные карточки:** Выручка с рейсов / Прочие поступления / Итого
- **Двухколоночный макет:** слева — таймлайн, справа — структура
- **Структура доходов (правая колонка):** StackBar + LegendRow + мини-карточки «Рейсы%» / «Поступления%»
- **Категории динамические** — строятся из реальных `income_transactions.category.name`
- **«Выручка с рейсов»** — агрегат из `trip_orders` (approved+completed), кликабельна → фильтр
- **Источник данных:** `GET /api/finance?month=YYYY-MM` → поля `income_transactions[]` и `revenue`

### Источники данных доходов

| Источник                  | Поле в ответе API       | Условие в БД                                                                      |
| ------------------------- | ----------------------- | --------------------------------------------------------------------------------- |
| Выручка с рейсов (агрег.) | `revenue`               | `trip_orders`: lifecycle=approved, settlement=completed, trips.lifecycle=approved |
| Ручные транзакции дохода  | `income_transactions[]` | `transactions`: type=income, lifecycle=approved, settlement=completed             |

## Панель «Дебиторка» (`recv`)

- Список должников с суммой и давностью долга
- Клик → список заказов должника → кнопка «Оплатить» (частично или полностью)
- Follow-up: статус / обещание заплатить / следующий звонок / заметка
- Добавить ручной долг
- Агрегация: `GET /api/receivables`

## Панель «Кредиты и лизинг» (`loans`)

- Список кредитов с остатком, ставкой, датой платежа
- Прогресс-бар погашения
- `GET /api/loans`

## Панель «Долги поставщикам» (`payables`)

- Три поставщика: Дерябин ГСМ / Новиков А.В. Запчасти / Ромашин Запчасти
- Долг = SUM(expense, settlement=pending, lifecycle=approved) по `counterparty_id`
- История транзакций по поставщику, кнопка «Погасить»
- `GET /api/payables`

## Учет транзакций

- **Типы:** Доход, Расход, Перевод.
- **Статусы:** Разделение на жизненный цикл (`draft` → `approved`) и факт оплаты (`pending` → `completed`).
- **Идемпотентность:** Защита от дублей — `idempotency_key` на каждой форме создания.
- **Балансы считаются только по:** `lifecycle_status='approved' AND settlement_status='completed'`

## Кошельки и балансы

Остаток кошелька вычисляется «на лету» из журнала транзакций (не хранится в поле).

- Расчётный счёт: `10000000-0000-0000-0000-000000000001`
- Сейф (Наличные): `10000000-0000-0000-0000-000000000002`
- Карта: `10000000-0000-0000-0000-000000000003`

## Управленческая отчетность

- **P&L (Прибыли и убытки):** Формирование финансового результата по категориям и направлениям.
- Выручка = `trip_orders` (approved + completed + trips.approved)
- Расходы = `transactions` (expense, approved, completed)
