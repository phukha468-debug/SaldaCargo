# WebApp: Ревью и Контроль

**Теги:** #webapp #контроль #финансы  
**Статус:** ✅ РЕАЛИЗОВАНО (09.05.2026)

## Суть

Раздел «Ревью» — ежедневная проверка рейсов администратором. До утверждения (`approved`) данные не попадают в P&L и балансы.

---

## Реализовано: `/review`

### DateNav — навигация по датам

- Тёмный блок `bg-slate-800`, высота `h-14`, скруглённый
- Крупные кнопки `←` / `→` (`w-40`, `bg-sky-500`), отключается при текущей дате
- По центру: иконка календаря + дата + день недели + кнопка «→ сегодня» если не сегодня
- Строка кнопок месяцев под навигацией (`bg-slate-700` / `bg-sky-500` активный)
- Клик на дату открывает нативный `<input type="date">` (скрытый через `sr-only`)

### TripCard — карточка рейса

**Свёрнутый вид (одна строка):**

- Левая часть (`flex-1 min-w-0`): эмодзи машины + название + водитель + номер + время
- Метрики (`shrink-0`, 3×`w-[100px]`): Выручка − Расходы = Прибыль (font-black, цветная)
- Кнопка редактирования + шеврон

**Развёрнутый вид:**

- Список заказов: тип, сумма, способ оплаты, ЗП
- Кнопки: Одобрить / Вернуть (с полем комментария)
- Удаление рейса: двухшаговое подтверждение — первый клик меняет цвет + animate-pulse, второй удаляет; таймер 4 сек сбрасывает если не подтвердил

**Важно:** метрики используют `shrink-0` (фиксированная ширина от правого края), левая часть — `flex-1 min-w-0`. Это обеспечивает выравнивание колонок между карточками.

### Суммарная строка за день

Тёмный блок `bg-slate-800` под карточками — те же `w-[100px]` колонки что и у карточек.  
Итоговые: выручка, расходы, прибыль (emerald/rose).

### EditModal — редактирование заказов

- Открывается по иконке редактирования на свёрнутой карточке
- Поля: сумма, ЗП водителя, ЗП грузчика, описание, способ оплаты
- `PATCH /api/trips/:id` с массивом orders

## API

### `GET /api/review?date=YYYY-MM-DD`

Возвращает рейсы за указанную дату (`trips.started_at`).  
Статусы: `lifecycle_status IN ('draft', 'approved', 'returned')` (не cancelled).  
Агрегирует выручку, расходы (trip_expenses + driver_pay + loader_pay) по каждому рейсу.

### `POST /api/review/approve` / `POST /api/review/return`

- **approve:** `lifecycle_status = 'approved'`, `settlement_status = 'completed'` для всех `trip_orders`
- **return:** `lifecycle_status = 'returned'`, обязательный `admin_note`

### `PATCH /api/trips/:id`

Редактирование полей заказов (`amount`, `driver_pay`, `loader_pay`, `description`, `payment_method`).

### `DELETE /api/trips/:id`

Мягкое удаление: `lifecycle_status = 'cancelled'`. Рейс перестаёт отображаться.
