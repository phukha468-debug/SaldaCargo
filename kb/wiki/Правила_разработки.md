# Правила разработки
**Теги:** #регламент #стандарты #workflow

Этот документ является обязательным к прочтению перед началом любой задачи. Нарушение этих правил ведет к отклонению PR.

## 1. Роли в проекте
| Роль | Субъект | Обязанности |
| :--- | :--- | :--- |
| **Заказчик** | Человек | Формулирует задачи, принимает решения. |
| **Архитектор** | Claude | Проектирует решения, создает задачи. |
| **Программист** | Gemini CLI | Исполняет задачи строго по спецификации. |

## 2. Кодекс «Никогда» (Taboos)
- 🚫 **Удаление данных:** Физическое удаление запрещено. Используется только `soft delete` (`status='cancelled'`).
- 🚫 **Деньги как Float:** Для денег используется только `DECIMAL(12,2)` в БД и `string` в коде.
- 🚫 **Секреты в коде:** Все ключи — только через `.env`.
- 🚫 **Бесхозный Async:** Каждый асинхронный вызов должен быть обернут в `try/catch`.
- 🚫 **Самодеятельность:** Не выходить за рамки текущей задачи и не менять архитектуру без согласования.

## 3. Стандарты именования
- **База данных:** `snake_case`, таблицы во множественном числе (напр. `transactions`).
- **Код (TS/React):** Компоненты — `PascalCase`, файлы — `kebab-case`.
- **UI:** Весь пользовательский интерфейс — на **русском языке**.

## 4. Финансовая дисциплина
- Балансы **никогда не хранятся** в базе как статические значения.
- Любой баланс вычисляется «на лету» из журнала [[Транзакции|транзакций]].
- Условия для расчета: `lifecycle_status='approved'` и `settlement_status='completed'`.

## 5. Протокол завершения задачи
1. Код работает без ошибок и предупреждений линтера.
2. Проведены все шаги верификации.
3. Заполнен `COMPLETION LOG`.
4. Файл задачи перемещен в `tasks/done/`.
