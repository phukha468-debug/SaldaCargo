## Что делает администратор

Администратор:

1. **Ревьюит рейсы** — одобряет или возвращает завершённые рейсы водителей
2. **Ведёт финансы** — вносит доходы и расходы вручную
3. **Управляет доступом** — добавляет/блокирует сотрудников в БД

## MiniApp: раздел `/admin/`

Все экраны доступны после выбора роли `admin` или `owner` на главном экране.
Нижний таб-бар: **Главная** / **Рейсы** / **Финансы**.

### Главная — `/admin`

Дашборд с живыми данными из `/api/admin/summary`:

| Карточка | Данные                                                      |
| -------- | ----------------------------------------------------------- |
| На ревью | Кол-во рейсов `status=completed AND lifecycle_status=draft` |
| В рейсе  | Кол-во активных рейсов `status=in_progress`                 |
| Сегодня  | Выручка из `trip_orders` одобренных за сегодня              |

- Оранжевый баннер: "X рейсов ждут ревью →" (ссылка на `/admin/trips?filter=review`)
- Быстрые кнопки: Доход / Расход (редирект на `/admin/finance?action=income/expense`)
- Кнопка выход в хедере

### Рейсы — `/admin/trips`

Три фильтра (кнопки):

- **На ревью** (`filter=review`) — завершены, ожидают одобрения, оранжевая рамка
- **Активные** (`filter=active`) — в рейсе прямо сейчас
- **Все** (`filter=all`) — последние 50 рейсов

Каждая карточка показывает:

- Номер рейса, машина, водитель, дата
- `LifecycleBadge` — статус жизненного цикла
- Выручка / ЗП / Расходы

### Детали рейса — `/admin/trips/{id}`

Страница полной информации:

- Машина + гос. номер, водитель, пробег
- Заметка водителя (если есть)
- Три карточки итогов: Выручка / ЗП / Расходы
- Список всех активных заказов (с контрагентом, способом оплаты, ЗП)
- Список расходов (категория, способ оплаты, сумма)

**Кнопки действий** (только для `lifecycle_status=draft`):

| Кнопка               | Действие                                                |
| -------------------- | ------------------------------------------------------- |
| ✓ Одобрить (зелёная) | `PATCH /api/admin/trips/{id}` с `action='approve'`      |
| ↩ Вернуть (жёлтая)   | Показывает textarea → `action='return', note='причина'` |

При одобрении:

- Рейс: `lifecycle_status → 'approved'`
- Все не-отменённые заказы: `lifecycle_status → 'approved'`
- Кэш `admin-trips` и `admin-summary` инвалидируется
- Редирект обратно на список

При возврате:

- Рейс: `lifecycle_status → 'returned'`, `driver_note = note`

### Финансы — `/admin/finance`

Два подменю (Доход / Расход), каждое открывает нужную форму.

**Добавить доход / расход (ручной ввод):**

- Выбор категории (плитки)
- Сумма
- Способ оплаты: Нал / Безнал / Карта
- Описание (необязательно)
- Создаётся с `lifecycle_status='approved'`, `settlement_status='completed'`

**Инкассация (подменю Доход → Инкасс.):**

- Выбор водителя
- Сумма
- Создаёт `transfer` транзакцию: подотчёт водителя → Сейф
- Все активные заказы `payment_method=cash` водителя — так попадают в Сейф

**Дебиторка (подменю Доход → Дебиторка) — `/api/admin/receivables`:**

- Список должников, сгруппированный по контрагенту
- Каждый заказ показывает тип долга: `📱 QR / Р/С`, `💳 На карту`, `⏳ Долг нал`
- При выборе заказа показывается целевой кошелёк: `→ 🏦 Расчётный счёт` / `→ 💳 Карта` / `→ 💵 Сейф`
- Кнопка "Погасить" → `POST /api/admin/receivables/settle`
- Сервер сам определяет нужный кошелёк по `payment_method` заказа

**Кредиты / Долги (подменю Расход → Кредиты):**

- Список активных кредитов и долгов поставщикам
- Оплата → expense транзакция из выбранного кошелька

**История:** последние 30 транзакций из таблицы `transactions`.

## Управление пользователями (через БД)

Пока нет UI — через Supabase Studio или SQL.

### Добавить нового сотрудника

```sql
INSERT INTO users (name, phone, max_user_id, roles, is_active)
VALUES ('Имя', '79001112233', 'max-id-здесь', ARRAY['driver']::user_role[], true);
```

### Заблокировать (уволить)

```sql
UPDATE users SET is_active = false WHERE id = 'uuid';
-- НЕ удалять запись — soft delete!
```

### Привязать MAX ID

Когда сотрудник войдёт в бота и пришлёт контакт:

```sql
UPDATE users SET max_user_id = 'полученный-id' WHERE name = 'Имя';
```

### Добавить машину

```sql
INSERT INTO assets (reg_number, short_name, asset_type_id, status)
VALUES ('А001АА196', 'Газель 001', 'uuid-типа', 'active');
```

## Известные ограничения MVP

- Нет WebApp (дашборд) — аналитика и финансы только в MiniApp
- Нет ролевой защиты в middleware — технически любой авторизованный может зайти на `/admin`
- Нет отчётов по периодам
- Нет управления сотрудниками через UI
