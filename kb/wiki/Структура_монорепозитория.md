# Структура монорепозитория

**Теги:** #архитектура #devops #pnpm #monorepo
**Обновлено:** 2026-05-07

Проект SaldaCargo — монорепозиторий на **pnpm workspaces**.

---

## Дерево проекта

```text
salda/
├── apps/
│   ├── miniapp/          # Next.js 15 — MiniApp для MAX мессенджера (основное приложение)
│   └── web/              # Next.js 15 — WebApp дашборд (в разработке)
├── packages/
│   ├── ui/               # @saldacargo/ui — UI-компоненты
│   ├── shared-types/     # @saldacargo/shared-types — типы и перечисления
│   └── domain/           # @saldacargo/domain-* — бизнес-логика (в разработке)
├── supabase/
│   ├── migrations/       # SQL-миграции (единственный источник истины схемы)
│   └── seed/             # Начальные данные
├── docs/                 # Техническая документация
│   ├── architecture/     # Архитектурные документы
│   ├── business/         # fleet.md, people.md — данные автопарка и персонала
│   ├── database/         # Схема БД
│   ├── design/           # HTML-макеты
│   ├── flows/            # Пользовательские сценарии
│   └── modules/          # Документация модулей domain
├── kb/                   # База знаний (Obsidian)
│   ├── wiki/             # Основные статьи (читать здесь)
│   └── raw/              # Сырые источники
├── tasks/
│   ├── todo/             # Задачи в работе
│   └── done/             # Выполненные задачи
└── scripts/              # Вспомогательные скрипты
```

## Пакеты (packages/)

### `@saldacargo/ui`

Путь: `packages/ui/src/index.ts`

Экспортирует:

- `<Money amount="5000.00" />` — форматирует деньги: `5 000 ₽`
- `<LifecycleBadge status="draft|approved|returned|cancelled" />` — бейдж статуса
- `<SettlementBadge status="pending|completed" />` — бейдж оплаты
- `<Button />`, `<Badge />`, `<NetworkIndicator />`
- `cn()` — утилита объединения Tailwind-классов

### `@saldacargo/shared-types`

Путь: `packages/shared-types/src/`

- `enums.ts` — все TypeScript-перечисления (LifecycleStatus, TripStatus, PaymentMethod, UserRole и т.д.)
- `models.ts` — интерфейсы сущностей
- `api.ts` — типы запросов/ответов API
- `database.types.ts` — авто-генерируемые типы Supabase (команда: `pnpm db:types`)

### `@saldacargo/domain`

Путь: `packages/domain/`
Бизнес-логика в модулях. Пока активно не используется в MiniApp (логика в API-роутах).

## MiniApp (`apps/miniapp/`)

```text
app/
├── page.tsx              # RootDispatcher — главный экран выбора роли / восстановления сессии
├── layout.tsx            # Корневой layout
├── middleware.ts         # Защита маршрутов по cookie
│
├── driver/               # Интерфейс водителя
│   ├── layout.tsx        # Bottom nav (Рейсы / Финансы / Профиль)
│   ├── page.tsx          # Главная водителя
│   ├── trips/page.tsx    # Список рейсов
│   ├── finance/page.tsx  # Финансы (подотчёт + ЗП)
│   └── profile/page.tsx  # Профиль, смена машины, выход
│
├── trip/[id]/            # Страница рейса
│   ├── page.tsx          # Детали рейса + удаление
│   ├── order/page.tsx    # Форма добавления заказа
│   ├── expense/page.tsx  # Форма добавления расхода
│   └── finish/page.tsx   # Завершение рейса (одометр)
│
├── mechanic/             # Интерфейс механика
│   ├── layout.tsx
│   ├── page.tsx
│   ├── orders/page.tsx
│   └── profile/page.tsx
│
├── admin/                # Интерфейс администратора
│   ├── layout.tsx        # Bottom nav (Главная / Рейсы / Финансы)
│   ├── page.tsx          # Дашборд (summary, быстрые действия)
│   ├── trips/
│   │   ├── page.tsx      # Список рейсов (фильтры: ревью/активные/все)
│   │   └── [id]/page.tsx # Детали рейса + одобрить/вернуть
│   └── finance/page.tsx  # Добавить доход/расход + история
│
└── api/                  # API-роуты (Next.js Route Handlers)
    ├── auth/             # max, logout, verify, users/route
    ├── trips/            # CRUD рейсов и вложенных заказов/расходов
    ├── driver/           # summary, finance, profile, assets, loaders
    ├── mechanic/         # orders, parts, summary
    ├── admin/            # summary, trips, transactions
    ├── users/public/     # Список пользователей (без авторизации)
    └── vehicles/public/  # Список машин (без авторизации)
```

## Ключевые файлы конфигурации

| Файл                                 | Назначение                            |
| ------------------------------------ | ------------------------------------- |
| `apps/miniapp/middleware.ts`         | Защита маршрутов по cookie            |
| `apps/miniapp/lib/supabase/admin.ts` | createAdminClient() с service_role    |
| `packages/shared-types/src/enums.ts` | Все TypeScript-перечисления           |
| `supabase/migrations/*.sql`          | Единственный источник истины схемы БД |
| `docs/business/fleet.md`             | Данные автопарка (источник истины)    |
| `docs/business/people.md`            | Данные персонала (источник истины)    |
| `tasks/todo/tt.md`                   | UUID категорий транзакций             |

## Команды разработки

```bash
# Запуск MiniApp
pnpm --filter miniapp dev

# Сборка
pnpm --filter miniapp build

# Обновить TypeScript-типы из схемы Supabase
pnpm db:types

# Запустить все приложения
pnpm dev
```
