# Структура монорепозитория
**Теги:** #архитектура #devops #pnpm #monorepo

Проект SaldaCargo организован как монорепозиторий на базе **pnpm workspaces**. Это позволяет хранить приложения и общие библиотеки в одном месте, обеспечивая строгую типизацию и переиспользование кода.

## Общий вид
```text
saldacargo/
├── apps/               # Приложения (deployable)
│   ├── web/            # Next.js: Админ-панель и аналитика
│   └── miniapp/        # Next.js: Мини-приложение для водителей (PWA/MAX)
├── packages/           # Переиспользуемые пакеты (internal)
│   ├── shared-types/   # Общие интерфейсы и типы БД
│   ├── ui/             # Библиотека UI-компонентов (shadcn/ui)
│   ├── api-client/     # Логика взаимодействия с Supabase
│   └── constants/      # Бизнес-константы и правила
├── supabase/           # Конфигурация БД и миграции
├── docs/               # Техническая документация
└── scripts/            # Вспомогательные скрипты (seed, types gen)
```

## Ключевые компоненты

### Приложения (apps/)
- **web:** Основной дашборд. Использует App Router (Next.js 15). Отвечает за тяжелые операции, отчеты и управление правами.
- **miniapp:** Оптимизирован под мобильные устройства и работу в мессенджере MAX. Поддерживает offline-режим через IndexedDB.

### Пакеты (packages/)
- **@saldacargo/ui:** Содержит все визуальные атомы. Основан на Tailwind и Radix UI.
- **@saldacargo/api-client:** Инкапсулирует вызовы к Supabase. Гарантирует, что бизнес-логика не «размазывается» по фронтенду.
- **@saldacargo/shared-types:** Центральное место для `database.types.ts`, генерируемых напрямую из схемы Supabase.

## Скрипты управления
В корне проекта доступны команды:
- `pnpm dev` — запуск всех приложений параллельно.
- `pnpm build` — сборка всего проекта.
- `pnpm db:types` — обновление типов TypeScript на основе текущей схемы БД.

## CI/CD
Проект настроен на автоматический деплой в Vercel через GitHub Actions. Скрипты деплоя настроены так, чтобы пересобирать только те части монорепозитория, в которых произошли изменения.
