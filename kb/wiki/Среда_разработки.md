# Среда разработки
**Теги:** #разработка #настройка #env #стек

Этот документ описывает техническое окружение проекта, необходимые инструменты и переменные среды.

## Технический стек
- **Runtime:** Node.js 18+
- **Framework:** Next.js 15 (App Router), React 19
- **Database/Auth:** Supabase
- **Package Manager:** pnpm

## Переменные окружения (.env)
Все чувствительные данные хранятся в файлах `.env.local` в соответствующих директориях приложений (`apps/web` и `apps/miniapp`).

| Переменная | Описание |
| :--- | :--- |
| `NEXT_PUBLIC_SUPABASE_URL` | Базовый URL проекта Supabase. |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Публичный ключ для клиентской части (соблюдает RLS). |
| `SUPABASE_SERVICE_ROLE_KEY` | **Секретный** ключ для сервера (обходит RLS). |
| `NEXT_PUBLIC_MAX_BOT_TOKEN` | Токен для интеграции с мессенджером MAX. |
| `OPTI24_API_KEY` | Ключ для синхронизации данных о топливе. |

## Работа с Supabase
В проекте предусмотрено три типа клиентов для разных уровней доступа:
1. **client.ts:** Используется в `use client` компонентах.
2. **server.ts:** Используется в серверных компонентах и API роутах.
3. **admin.ts:** Использует `SERVICE_ROLE_KEY`. Доступен **только** на сервере для системных операций.

## Основные команды (pnpm)
- `pnpm install` — установка зависимостей во всех пакетах.
- `pnpm dev:web` — запуск дашборда (localhost:3000).
- `pnpm dev:miniapp` — запуск мини-приложения (localhost:3001).
- `pnpm build` — сборка всех приложений и пакетов.

## Ограничения и лимиты
Проект оптимизирован под бесплатные лимиты **Vercel Hobby** и **Supabase Free Tier**.
- Для фоновых задач (Cron) используется расширение `pg_cron` внутри Supabase, так как в Vercel Hobby кроны ограничены.
- Объем базы данных ограничен 500 МБ.
