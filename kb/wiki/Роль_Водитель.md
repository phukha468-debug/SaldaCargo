# Роль: Водитель

**Теги:** #роли #miniapp #логистика
**Обновлено:** 2026-05-07

## Суть

Водитель — основной поставщик данных «с полей». Работает только в MiniApp (мессенджер MAX).
Создаёт данные в статусе `draft`. Администратор одобряет или возвращает.

---

## Экраны MiniApp (`/driver/`)

### Вход в систему

- Маршрут: `/` (RootDispatcher)
- Выбирает роль → выбирает себя из списка → выбирает машину
- Сессия сохраняется в cookie (`salda_user_id`) и `localStorage` (`selected_role`, `active_vehicle_id`)
- При перезапуске/смахивании — сессия восстанавливается автоматически (cookie + localStorage)

### Главная — `/driver`

- Вкладки: **Рейсы** / **Финансы** / **Профиль** (нижний таб-бар)
- Показывает: активный рейс (если есть) или кнопку "Начать рейс"

### Вкладка Рейсы — `/driver/trips`

- Список последних рейсов водителя
- Статус жизненного цикла (`LifecycleBadge`): draft / approved / returned / cancelled
- Нажатие на рейс → страница деталей

### Страница рейса — `/trip/{id}`

- Детальная информация: машина, дата, список заказов и расходов
- **Для активного рейса** (`in_progress`):
  - Кнопка "Добавить заказ" → `/trip/{id}/order`
  - Кнопка "Добавить расход" → `/trip/{id}/expense`
  - 🗑️ иконки с подтверждением: удалить заказ (отмена) / удалить расход (физически)
  - Кнопка "Завершить рейс" → `/trip/{id}/finish`
- После добавления заказа/расхода — страница обновляется мгновенно

### Добавить заказ — `/trip/{id}/order`

Поля формы:

- Клиент (контрагент из списка или новый)
- Сумма (DECIMAL)
- ЗП водителя (вводится вручную, подсказка ~30%)
- ЗП грузчика (вводится вручную)
- Способ оплаты: наличные / QR / безнал / долг наличными / карта водителя
- Описание (необязательно)

Защита от двойного сабмита: `submitted` state блокирует повторный вызов.

### Добавить расход — `/trip/{id}/expense`

Поля:

- Категория расхода (из `transaction_categories`)
- Сумма
- Способ оплаты: cash / card_driver / fuel_card
- Описание (необязательно)

### Завершить рейс — `/trip/{id}/finish`

- Поле ввода одометра (конец)
- При фокусе — поле скроллится вверх (над клавиатурой)
- Кнопка внутри формы (не фиксированная), чтобы клавиатура не перекрывала
- Заметка водителя (необязательно)

### Вкладка Финансы — `/driver/finance`

Два раздела:

**Подотчёт** (нал на руках у водителя):

- Считается СРАЗУ из `trip_orders` с `payment_method IN ('cash', 'card_driver')`
- Не ждёт одобрения от администратора
- Показывает список наличных заказов как историю

**ЗП за месяц:**

- Считается из утверждённых (`lifecycle_status='approved'`) рейсов текущего месяца
- Суммирует `trip_orders.driver_pay`

### Вкладка Профиль — `/driver/profile`

- Имя водителя, телефон
- Текущая машина с кнопкой "Сменить" (открывает пикер)
- Выход (очищает cookie + localStorage)

## Правила ЗП (напоминание)

**ЗП вводится ВРУЧНУЮ при каждом заказе.** Система показывает подсказки, но не считает автоматически.

| Тип рейса             | Примерная ставка                    |
| --------------------- | ----------------------------------- |
| Газель (город)        | ~30% от суммы заказа                |
| Газель (межгород Екб) | ~25% или фикс ~3 000 ₽              |
| Переезды              | Почасовая 3 000 ₽/час, 33% водителю |
| Валдай (город)        | ~30%                                |
| Валдай (дальний)      | 10 ₽/км + 1 000 ₽/сутки             |

## Статусы рейса — что видит водитель

| status      | lifecycle_status | Что значит                              |
| ----------- | ---------------- | --------------------------------------- |
| in_progress | draft            | Рейс активен, можно добавлять заказы    |
| completed   | draft            | Отправлен на ревью, ждёт администратора |
| completed   | approved         | Одобрен ✅                              |
| completed   | returned         | Возвращён ❌ — нужно исправить          |
| cancelled   | cancelled        | Отменён                                 |

## Связанные API-маршруты

| Действие        | Метод     | URL                                  |
| --------------- | --------- | ------------------------------------ |
| Список рейсов   | GET       | /api/driver/trips                    |

| Отменить заказ  | PATCH     | /api/trips/{id}/orders/{orderId}     |

| Удалить расход  | DELETE    | /api/trips/{id}/expenses/{expenseId} |
| Завершить рейс  | POST      | /api/trips/{id}/finish               |
| Финансы         | GET       | /api/driver/finance                  |
| Профиль         | GET/PATCH | /api/driver/profile                  |
