# Модель данных
**Теги:** #бд #sql #структура

## Обзор
База данных SaldaCargo построена на PostgreSQL (Supabase) и спроектирована для обеспечения целостности финансовых потоков и истории изменений.

## Ключевые сущности
- **[[Транзакции]] (`transactions`):** Центральная таблица. Любое движение денег фиксируется здесь.
- **Активы (`assets`):** Транспортные средства и оборудование. Хранят данные об [[Одометр|одометре]] и амортизации.
- **Рейсы (`trips`):** Объединяют заказы (`trip_orders`) и экипаж.
- **Кошельки (`wallets`):** Места хранения денег (касса, р/с, подотчеты сотрудников).
- **Сотрудники (`users`):** Ролевая модель доступа и привязка к MAX ID.

## Особенности реализации
- **RLS (Row Level Security):** Политики доступа прописаны в SQL миграциях. Водитель физически не может прочитать чужие рейсы.
- **Триггеры:** Автоматическое создание подотчетных кошельков при добавлении сотрудников, генерация транзакций на основе заказов, проверка регламентов ТО.
- **Индексы:** Оптимизированы для фильтрации по статусам (`lifecycle_status`) и датам.
