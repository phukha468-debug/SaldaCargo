# SaldaCargo WebApp Logic v1.0
**Тип:** техническое задание / описание логики
**Дата добавления:** 2026-04-29
**Источник:** [[raw/SaldaCargo_WebApp_Logic_v1.md]]

## Ключевые идеи
- **Центр управления:** WebApp является «мозгом» системы, где происходит апрув данных, глубокая аналитика и управление справочниками.
- **Ролевой UI:** Интерфейс динамически адаптируется под роли (`owner`, `admin`, `accountant`, `mechanic_lead`).
- **Юнит-экономика:** Каждая машина рассматривается как отдельный центр прибыли с расчетом прибыли на 1 км пробега.
- **Точка контроля (Ревью):** Процесс перевода данных из `draft` в `approved` как обязательный этап финансового учета.
- **Realtime-мониторинг:** Подписки на таблицы для мгновенного отображения новых рейсов и нарядов, требующих внимания.

## Противоречия и вопросы
- Необходимость ручного подтверждения каждого рейса (риск возникновения «бутылочного горлышка» при росте парка).
- Зависимость от внешних API (Wialon, Опти24) для валидации данных о пробеге и ГСМ.
