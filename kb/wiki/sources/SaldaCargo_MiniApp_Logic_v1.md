# SaldaCargo MiniApp Logic v1.0
**Тип:** техническое задание / описание логики
**Дата добавления:** 2026-04-29
**Источник:** [[raw/SaldaCargo_MiniApp_Logic_v1.md]]

## Ключевые идеи
- **Ролевая модель:** Четкое разделение функционала MiniApp для Водителя, Механика и Админа.
- **Офлайн-первичность:** Использование IndexedDB и `idempotency_key` для надежной работы в условиях плохой связи.
- **Двухосный статус:** Разделение жизненного цикла (`lifecycle`) и финансового завершения (`settlement`) транзакций.
- **Водитель как регистратор:** Ручной ввод данных о рейсе (ЗП, одометр, заказы) с последующим подтверждением админом.
- **Админ-пульт:** Возможность быстрой инкассации и управления денежными потоками через мобильное приложение.

## Противоречия и вопросы
- Необходимость ручного ввода ЗП водителем (потенциал для ошибок, требует тщательного ревью).
- Ручная проверка одометров по фото (отсутствие автоматической валидации на этапе MVP).
