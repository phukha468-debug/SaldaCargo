# Интеграция авторизации через МАКС (Verified Contact)

**Тип:** задача / спецификация
**Дата добавления:** 2026-04-30
**Источник:** [[raw/TASK_MAX_AUTH_INTEGRATION.md]]

## Ключевые идеи
- **Беспарольный вход:** Использование верифицированного контакта из мессенджера МАКС.
- **Безопасность:** Проверка подлинности данных через HMAC-SHA256 (Hash-based Message Authentication Code).
- **Связка данных:** Авторизация разрешена только если номер телефона из МАКС совпадает с номером в таблице `users` в БД.

## Процесс настройки (Manual Steps)
1. **Бот:** В мессенджере МАКС бот должен быть настроен на отправку кнопки `request_contact`. Это системная кнопка мессенджера, которая инициирует передачу верифицированного номера телефона.
2. **Токен:** `MAX_BOT_TOKEN` должен быть сохранен в переменных окружения (`.env`) и использоваться как секретный ключ для HMAC.
3. **Приложение:** В настройках бота кнопка `open_app` должна указывать на URL MiniApp.

## Механика верификации (Flow)
1. Пользователь нажимает системную кнопку `request_contact` в чате с ботом.
2. МАКС формирует VCF-структуру с номером телефона и подписывает её HMAC-SHA256, используя токен бота.
3. Данные (`vcf_info`, `hash`) отправляются на эндпоинт `/api/auth/verify`.
4. Сервер проверяет подпись. Если она валидна — значит номер телефона действительно принадлежит этому аккаунту МАКС (верифицирован мессенджером).
5. Только после криптографической проверки сервер ищет этот телефон в таблице `users`.

## Управление пользователями
- **Добавление:** Чтобы новый пользователь мог войти, его номер телефона должен быть предварительно внесен в таблицу `users`.
- **Лишение доступа:** Изменение статуса пользователя в БД или его удаление/блокировка мгновенно прекращает возможность авторизации, так как API `verify` проверяет наличие активной записи.
