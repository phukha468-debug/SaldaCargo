## ENUMS (типы-перечисления)

```sql
lifecycle_status  → 'draft' | 'approved' | 'returned' | 'cancelled'
settlement_status → 'pending' | 'completed'
trip_status       → 'in_progress' | 'completed' | 'cancelled'
trip_type         → 'local' | 'intercity' | 'moving' | 'hourly'
payment_method    → 'cash' | 'qr' | 'bank_invoice' | 'debt_cash' | 'card_driver'
transaction_direction → 'income' | 'expense' | 'transfer'
user_role         → 'owner' | 'admin' | 'driver' | 'loader' | 'mechanic' | 'mechanic_lead' | 'accountant'
asset_status      → 'active' | 'repair' | 'reserve' | 'sold' | 'written_off'
wallet_type       → 'bank_account' | 'cash_register' | 'fuel_card' | 'driver_accountable' | 'owner_personal' | 'company_card'
defect_urgency    → 'critical' | 'soon' | 'low'
defect_status     → 'open' | 'planned' | 'fixed' | 'dismissed'
counterparty_type → 'client' | 'supplier' | 'both'
service_order_machine_type → 'own' | 'client'
```

### `trips`

| Поле                       | Тип                               | Описание                                 |

| `odometer_start`           | INTEGER NOT NULL                  | Одометр в начале                         |
| `odometer_end`             | INTEGER                           | Одометр в конце (NULL пока рейс активен) |

| `approved_at`              | TIMESTAMPTZ                       |                                          |

| `ended_at`                 | TIMESTAMPTZ                       |                                          |
| `created_at / updated_at`  | TIMESTAMPTZ                       |                                          |

**Жизненный цикл рейса:**

1. Создаётся водителем → `status='in_progress'`, `lifecycle_status='draft'`
2. Водитель завершает → `status='completed'`, `lifecycle_status='draft'`
3. Админ одобряет → `lifecycle_status='approved'`
4. Админ возвращает → `lifecycle_status='returned'` (водитель должен исправить)

**Ревью-фильтр:** `status='completed' AND lifecycle_status='draft'`

## RLS (Row Level Security)

| Таблица          | Политика                                                  |
| ---------------- | --------------------------------------------------------- |
| `users`          | SELECT: все (USING true)                                  |
| `assets`         | SELECT: все (USING true)                                  |
| `trips`          | SELECT: свои (driver_id) или admin/owner                  |
| `trip_orders`    | SELECT: через свои trips                                  |
| `trip_expenses`  | SELECT: через trips.driver_id; DELETE: через admin client |
| `transactions`   | admin/owner — полный доступ; driver — только свои         |
| `service_orders` | механик — свои; admin/owner — все                         |

> **Важно:** все API-роуты в MiniApp используют `createAdminClient()` (service_role key) — обходят RLS.
> GRANT на DELETE для `trip_expenses` выдан вручную: `GRANT SELECT, INSERT, UPDATE, DELETE ON ALL TABLES IN SCHEMA public TO anon, authenticated, service_role;`
