### Шаг 5: Возьми РОВНО ОДНУ задачу из `tasks/todo/`

## Правила работы

1. **Одна задача за раз.** Не бери вторую, пока не закончил первую.
2. **Не трогай файлы, не указанные в задаче.** Никогда.
3. **Не додумывай.** Если задача неясна — остановись и спроси Заказчика.
4. **Не принимай стратегических решений.** Ты исполнитель, не архитектор.
5. **Не хардкодь секреты.** Токены, пароли, ключи — только через переменные окружения.
6. **Не оставляй мусор.** Никаких console.log, закомментированного кода, TODO без задачи.
7. **[v4.1] Try/catch обязателен.** Каждый async-вызов, каждый API-роут, каждый webhook-обработчик — в try/catch с console.error и полным контекстом ошибки.
8. **[v4.1] Await перед навигацией.** Запрещено выполнять router.push(), setState() или переход на следующий экран до завершения await записи в БД.

## Протокол завершения задачи (v4.1)

1. ✅ Код работает без ошибок
2. ✅ Выполнены ВСЕ шаги верификации из таска (конкретные проверки, не «запусти и посмотри»)
3. ✅ (UI-таски) `/i-audit` пройден, P0 и P1 исправлены
4. 📋 **[v4.1] Заполнен COMPLETION LOG** в конце файла задачи (обязательно!)
5. 📁 Файл задачи перемещён из `tasks/todo/` в `tasks/done/`
6. 📝 Статус обновлён в `docs/BRIEF.md` (отметить выполненный пункт)

### Что входит в COMPLETION LOG

Заполни секцию `## COMPLETION LOG` в конце файла задачи:
- **Статус:** completed / partial / blocked
- **Что сделано:** фактические действия (могут отличаться от постановки)
- **Изменённые файлы:** каждый файл + что изменено (1 строка)
- **Верификация:** результаты проверок (build, линтер, ручное тестирование)
- **Побочные эффекты:** что может сломаться в результате твоих изменений
- **Открытые вопросы:** нерешённые проблемы для follow-up

**Таск БЕЗ заполненного COMPLETION LOG не считается завершённым.**

## Протокол при ошибке

1. **НЕ чинить архитектуру самостоятельно.**
2. Описать проблему Заказчику: что пытался сделать, какая ошибка, почему не получается.
3. Оставить задачу в `tasks/todo/` (НЕ перемещать в done).
4. Дождаться решения Заказчика.

## [v4.1] Чек-лист приёмки для Заказчика

> Заказчик (Человек): перед словами «ОК, следующий таск» проверь:

```
□ COMPLETION LOG заполнен?
□ Верификация выполнена (есть скриншот/лог/результат)?
□ «Побочные эффекты» пустые или приемлемые?
□ «Открытые вопросы» пустые или запланированы?
□ Build проходит без ошибок?
```

**Если хоть один пункт провален — таск НЕ принят. Создай таск на доработку.**
