## Процесс / Методология

### LL-PROC-001: Completion Log обязателен

| Поле | Значение |
|------|----------|
| **Проект** | EVA (ретроспектива) |
| **Проблема** | 129/129 задач без лога завершения |
| **Последствие** | Невозможно восстановить контекст, потеря информации между сессиями |

**Правило:** Каждый таск содержит секцию COMPLETION LOG. Перенос в done/ без заполненного лога — запрещён.

### LL-PROC-002: «Финальный» таск — не финальный

| Поле | Значение |
|------|----------|
| **Проект** | EVA |
| **Проблема** | 5 тасков назывались «финальными», после каждого — ещё один «финальный» |
| **Первопричина** | Верификация была формальной: «проверь запуском» без конкретных критериев |

**Правило:** В каждом таске — конкретные шаги верификации (что запустить, что ожидать, что считать успехом). Слово «финальный» в названии запрещено.

### LL-PROC-003: Chain Breaker при 3-м фиксе

| Поле | Значение |
|------|----------|
| **Проект** | EVA |
| **Проблема** | Цепочки rework: 9 задач (подписки), 10+ задач (рефералы) |
| **Первопричина** | Каждый фикс лечил симптом, не причину |

**Правило:** 3-й фикс одной подсистемы → СТОП → ADR (Architecture Decision Record) → один таск-оверхол вместо серии патчей.
