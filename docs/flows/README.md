# Что такое flows

Flow — это пользовательский сценарий, описанный end-to-end: кто действует, что делает в UI, что происходит технически, какие данные летят между MiniApp, WebApp и БД.

Flows — главный документ для понимания «как это всё работает в жизни». Когда AI-ассистент берётся за задачу — он сначала смотрит соответствующий flow и только потом пишет код.

## Шаблон документа flow

Каждый flow следует одной структуре:

1. **Контекст** — кто действует, зачем, как часто.
2. **Шаги сценария** — пошагово, с UI и техническим описанием.
3. **Точки отказа** — что может пойти не так.
4. **Что важно знать для реализации** — нюансы для разработчика.
5. **Связанные документы** — модули, схемы, другие flows.

## Когда обновлять flow

- Меняется UI → обновляешь шаги.
- Меняется API → обновляешь технические описания.
- Появляется новая роль или новый сценарий → создаёшь новый файл.

Нельзя оставлять flow устаревшим. Если документ расходится с реальностью — он хуже, чем его отсутствие.
