# 🛠 ЗАДАЧА: Починка списка пользователей и аудит MAX-авторизации

## 1. Контекст и Роль
**Роль:** Senior Fullstack Developer.
**Проект:** SaldaCargo (MiniApp).
**Проблема 1:** При выборе роли на странице логина список пользователей пуст ("Пользователи не найдены").
**Проблема 2:** Приложение при открытии в МАХ игнорирует попытку авторизации через МАХ и сразу показывает fallback-экран выбора роли.

## 2. Строгие правила
1. **NO REFACTORING:** Не переписывай логику UI компонентов.
2. **USE ADMIN CLIENT:** Для получения списка пользователей на экране логина используй `createAdminClient()`, чтобы избежать блокировок RLS.
3. **ARRAY QUERY:** В БД `roles` — это массив. Используй правильный метод Supabase (`.contains('roles', [role])` или `.cs()`) для фильтрации.

## 3. Пошаговое выполнение

**Шаг 1: Почини выдачу пользователей (`/api/auth/users/route.ts`)**
- Найди или создай файл API, который обрабатывает `GET /api/auth/users`.
- Используй `createAdminClient()`.
- Напиши корректный запрос к таблице `users`, который ищет наличие переданной роли в массиве `roles` и где `is_active = true`.
- Внеси изменения в этот файл.

**Шаг 2: Покажи логику инициализации МАХ**
- Найди файл, в котором происходит попытка авторизации через МАХ (обычно это происходит в `useEffect` внутри `app/page.tsx`, `app/layout.tsx` или в отдельном хуке авторизации).
- **НЕ ИЗМЕНЯЙ ЭТИТ ФАЙЛ.** Просто выведи в отчет его текущий код (или ту часть, где проверяется среда МАХ и делается запрос к `/api/auth/max`), чтобы мы могли проанализировать, почему процесс пропускается.
