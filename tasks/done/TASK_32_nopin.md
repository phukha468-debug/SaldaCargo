# 🛠 ЗАДАЧА 32: Упрощенный вход (Selector Flow) и полный обход авторизации

## 1. Контекст и Цель
**Проблема:** Текущая система безопасности (MAX ID, ПИН-коды, RLS) блокирует процесс разработки бизнес-логики.
**Задача:** Временно отключить строгую проверку ПИН-кодов и автоматический вход. Реализовать простой визуальный выбор пользователя ("Витрина"), чтобы любой мог зайти под любой ролью для тестирования интерфейсов.

## 2. Жесткие ограничения (DO NOT TOUCH)
1. **НЕ ТРОГАЙ** структуру папок `app/driver`, `app/mechanic`, `app/admin`.
2. **НЕ ТРОГАЙ** стилистику: используй уже имеющийся дизайн (Zinc/Slate, кнопки с закруглением `rounded-3xl`, акценты Orange).

## 3. Техническое требование (Bypass RLS)
Для того чтобы список пользователей грузился у всех, необходимо:
1. Создать или обновить API роут `/api/users/public`, который использует `createAdminClient` (supabaseServiceRole), чтобы игнорировать политики RLS.
2. API должен возвращать массив объектов: `{ id, full_name, roles }`.

## 4. Пошаговый алгоритм реализации (UI Flow)

Трансформируй `app/page.tsx` в пошаговый мастер (Wizard):

### Шаг 1: Выбор роли
- Первый экран, который видит ЛЮБОЙ пользователь.
- Три большие кнопки: **"🚛 ВОДИТЕЛЬ"**, **"🔧 МЕХАНИК"**, **"👑 АДМИНИСТРАЦИЯ"**.

### Шаг 2: Выбор ФИО
- После выбора роли показывается список пользователей из БД, у которых в массиве `roles` есть выбранная роль.
- **Формат отображения:** "Фамилия И.О." (например, Шахмаев А.О.). 
- ПИН-код НЕ запрашивать. При клике на имя — пользователь считается выбранным.

### Шаг 3: Выбор машины (ТОЛЬКО для роли "Водитель")
- Если выбрана роль "Водитель", после выбора ФИО показать список доступных машин из таблицы `vehicles`.
- После выбора машины — переход в интерфейс.

### Шаг 4: Финализация входа
- При выборе пользователя (и машины):
  1. Записать `id` пользователя в куку `salda_user_id` (для совместимости с Middleware).
  2. Если это водитель — сохранить ID выбранной машины (например, в `localStorage` или временную куку `active_vehicle_id`).
  3. Выполнить `router.push()` в соответствующий раздел (`/driver`, `/mechanic`, `/admin`).

## 5. Ожидаемый результат
- При открытии приложения пользователь за 2-3 клика попадает в рабочий интерфейс без ввода ПИН-кода и проверок MAX ID.
- Список пользователей корректно подгружается благодаря использованию `adminClient`.
- Код чистый, без лишних `useEffect` на странице логина.
- `pnpm build` проходит успешно.
