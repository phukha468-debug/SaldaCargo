# 🛠 ЗАДАЧА 23: Создание экрана саморегистрации (получения MAX ID)

## 1. Контекст и Роль
**Роль:** Senior Fullstack Developer.
**Проблема:** Водителям неудобно узнавать свой MAX ID через ошибки. Нам нужен цивилизованный способ.
**Цель:** Если пользователь зашел через МАХ, но его ID нет в базе, показать красивый экран с его ID и кнопкой для копирования.

## 2. Строгие правила
1. **NO REFACTORING:** Не меняй логику парсинга URL из Task 22.
2. **UI/UX:** Экран должен выглядеть профессионально, в стиле SaldaCargo.

## 3. Пошаговое выполнение

**Шаг 1: Модифицируй обработку 403 ошибки в `app/page.tsx`**
- Если API `/api/auth/max` вернуло 403 и в сообщении есть "Ваш MAX ID: ...", не перенаправляй на логин.
- Вместо этого отобрази на весь экран блок с информацией:
  - Заголовок: "Вы еще не зарегистрированы"
  - Текст: "Передайте этот ID администратору для активации вашего аккаунта:"
  - Поле с номером ID (крупно и четко).
  - Кнопка "Скопировать ID".

**Шаг 2: Добавь кнопку "Зайти как гость"**
- Под блоком с ID добавь кнопку "Ввести ПИН-код вручную" (редирект на `/login`), если вдруг водителю нужно зайти под временным паролем.

## 4. Ожидаемый результат
Diff файла `page.tsx` с новым интерфейсом экрана регистрации.
