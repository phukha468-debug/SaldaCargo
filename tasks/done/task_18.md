# 🛠 ЗАДАЧА Восстановление автоматической авторизации МАХ

## 1. Контекст и Роль

Роль Senior Fullstack Developer.
Проблема Приложение на фронтенде игнорирует среду МАХ и сразу показывает экран выбора ролей. Нам нужно, чтобы оно автоматически пыталось авторизоваться через МАХ.
Цель Восстановить вызов `apiauthmax` при инициализации приложения.

## 2. Строгие правила

1. NO REFACTORING Не меняй логику выбора ролей и ПИН-кода.
2. MINIMAL CHANGES Только восстанови логику инициализации МАХ в `apppage.tsx`.

## 3. Целевые файлы

- `appsminiappapppage.tsx` (или файл, который ты аудировал как основной входной компонент).

## 4. Пошаговое выполнение

Шаг 1 Восстанови вызов МАХ

- В основном компоненте (в `useEffect`) добавь логику, которая извлекает данные из `window.MAX` (или аналогичного SDK) и отправляет POST-запрос на `apiauthmax`.
- Убедись, что если запрос возвращает 403 с нашим новым сообщением об ошибке (где указан MAX ID), это сообщение корректно отображается на экране.

Шаг 2 Проверь локально

- Убедись, что если данных МАХ нет (в обычном браузере), приложение по-прежнему показывает экран выбора ролей.

## 5. Ожидаемый результат

1. Diff изменений в `apppage.tsx`.
2. Краткое описание как именно теперь срабатывает проверка МАХ.
