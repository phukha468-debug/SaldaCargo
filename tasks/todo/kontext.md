## 1. Что за проект

SaldaCargo — ERP-система для транспортной компании (Верхняя Салда).  
Цель: в будущем продать как коробочное решение для других транспортных компаний.

**Два приложения:**

- `apps/miniapp` — Next.js 15, мобильный интерфейс для водителей/механиков/админа (мессенджер MAX)
- `apps/web` — Next.js 15, десктопный дашборд для владельца и админа

**Стек:** Next.js 15, React 19, TypeScript, Supabase (PostgreSQL), TanStack Query, Tailwind CSS 4  
**Auth:** кастомная cookie-схема (`salda_user_id`), НЕ Supabase Auth  
**Деплой:** Vercel (miniapp), локально (webapp пока тестируется)  
→ Подробнее: [`kb/wiki/Технический_стек.md`](../../kb/wiki/Технический_стек.md)

## 2. Что уже реализовано

### MiniApp (apps/miniapp) — ГОТОВО ✅

- Авторизация водителей/механиков/админа через MAX и PIN
- **Водитель:** создание рейса, заказы (способы оплаты), расходы, завершение с одометром
- **Механик:** наряды, запчасти, статусы
- **Админ (мобильный):**
  - Дашборд с KPI (summary)
  - Список рейсов с фильтрами (ревью / активные / все)
  - Одобрить / Вернуть рейс (с отображением ошибок)
  - Финансы: добавить доход/расход по категориям
  - **Инкассация:** видит подотчёт каждого водителя, проводит сбор наличных

### WebApp (apps/web) — В РАЗРАБОТКЕ 🔨

- Авторизация (PIN, отключена для локального тестирования)
- **Главная (/):** KPI за месяц из реальных данных БД (выручка, расходы, прибыль, сегодня, рейсы на ревью, последние транзакции)
- **Ревью (/review):** одобрение/возврат рейсов + режим История по датам
  - Дизайн: тёмный DateNav (slate-800) с крупными sky-500 кнопками, компактные карточки
  - Метрики выровнены по колонкам (shrink-0 w-[100px]), суммарная строка за день
  - Удаление рейса: двухшаговое подтверждение с таймером 4 сек
- **Ретро (/retro):** ручной ввод рейса задним числом
- **Финансы (/finance):** P&L за 6 месяцев, журнал транзакций с фильтром, подотчёт водителей
- **Дебиторка (/receivables):** список должников, сумма, просрочка, отметить оплаченным
- **Автопарк (/fleet):** ✅ ГОТОВО (09.05.2026)
  - Компактные коллапсируемые плитки машин (grid 3 колонки, items-start)
  - Полная юнит-экономика: операционные расходы + ТО (запчасти из СТО) + постоянные
  - Три уровня прибыли: операционная, реальная (true_profit), маржа %
  - Себестоимость 1 км: операционная и полная (true_cost_per_km)
  - Счётчики рейсов, заказов, средний чек, средний пробег на рейс
  - Красный баннер для убыточных машин
  - Поле `monthly_fixed_cost` (ОСАГО + налог + лизинг) в форме редактирования

## 4. Данные нужные от пользователя

### 4.1 Кредиты — параметры для каждого кредита:

```
1. Кредитор           — название банка или организации (пр: «Сбербанк», «ВТБ Лизинг»)
2. Тип                — кредит / лизинг / займ
3. Назначение         — на что брался (пр: «Валдай 2022», «оборотные средства»)
4. Исходная сумма     — сколько взяли изначально (в рублях)
5. Остаток на сегодня — сколько ещё осталось выплатить
6. Ставка             — процент годовых (пр: 14.5%)
7. Ежемесячный платёж — фиксированная сумма в месяц
8. Дата начала        — когда взяли (пр: 15.03.2023)
9. Дата окончания     — когда закрывается (пр: 15.03.2028)
10. Примечание        — любая доп. информация (необязательно)
```

### 4.2 Существующие должники — параметры для каждого:

```
1. Название организации  — (пр: «ООО Агрострой»)
2. Телефон               — (необязательно)
3. Сумма долга           — в рублях
4. Дата возникновения    — примерная дата (пр: 25.04.2026)
5. За что                — описание (пр: «Рейс Салда-Екб, 2 машины»)
6. Тип оплаты            — «безнал» или «долг наличными»
```

### 4.3 Постоянные расходы на машины (`monthly_fixed_cost`):

Заполняется через UI `/fleet` → кнопка «Изменить» на каждой машине.  
Поле: **«Постоянные расходы в месяц (₽)»** = ОСАГО/12 + транспортный налог/12 + платёж по лизингу/кредиту.  
Пример: ОСАГО 36 000 в год = 3 000/мес. Налог 15 000/год = 1 250/мес.

## 5. Ключевые технические факты

- `users.roles` — **массив** `user_role[]`, фильтровать через `.contains('roles', ['driver'])`, НЕ `.eq('role', ...)`
- Все API-роуты webapp используют `createAdminClient()` из `apps/web/lib/supabase/admin.ts`
- `as any` ставится на `.from('table')`: `(supabase.from('trips') as any).select(...)`
- Деньги — только строкой: `"5000.00"`, компонент `<Money amount="5000.00" />`
- Инкассации — таблица `cash_collections` (создана 07.05.2026, миграция применена)
- Auth webapp отключена для локального тестирования (`apps/web/middleware.ts`)
- Грузчики — **не персонализированные сущности**, просто `loaders_count SMALLINT` на `trips` + `loader_pay`/`loader2_pay` на `trip_orders`. Нет FK на users.
- Оптимистичная навигация в MiniApp: `router.push()` сразу, fetch в фоне — интерфейс мгновенный.
- `staleTime` на trip detail: 20000мс (не 0) — убирает лишние запросы при навигации.
- CSS Grid для плиток: всегда добавлять `items-start` — иначе все карточки в строке растягиваются до высоты самой высокой (баг раскрытия соседних тайлов).
- Генерация TypeScript-типов из удалённого Supabase: `npx supabase gen types typescript --project-id dzwuvbhfqnsxbfbynmgz > packages/shared-types/src/database.types.ts`
