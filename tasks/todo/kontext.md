## 1. Что за проект

SaldaCargo — ERP-система для транспортной компании (Верхняя Салда).  
Цель: в будущем продать как коробочное решение для других транспортных компаний.

**Два приложения:**

- `apps/miniapp` — Next.js 15, мобильный интерфейс для водителей/механиков/админа (мессенджер MAX)
- `apps/web` — Next.js 15, десктопный дашборд для владельца и админа

**Стек:** Next.js 15, React 19, TypeScript, Supabase (PostgreSQL), TanStack Query, Tailwind CSS 4  
**Auth:** кастомная cookie-схема (`salda_user_id`), НЕ Supabase Auth  
**Деплой:** Vercel (miniapp и webapp — оба задеплоены)

## 3. Архитектура учёта ЗП (важно!)

ЗП — **два разных источника**, не смешивать:

| Источник                                            | Что даёт                 | Где используется                                   |
| --------------------------------------------------- | ------------------------ | -------------------------------------------------- |
| `trip_orders.driver_pay / loader_pay / loader2_pay` | Начисленная ЗП из рейсов | P&L, плитка ЗП на дашборде, "заработано" на /staff |
| `transactions` (category PAYROLL\_\*)               | Факт выплаты денег       | "выплачено" на /staff, debt tracking               |

**Правило:** PAYROLL-транзакции **исключены** из P&L (`expenses`), чтобы не задваивать с `payrollFromOrders`.  
Это реализовано в:

- `apps/web/app/api/dashboard/summary/route.ts` — `.not('category_id', 'in', PAYROLL_IDS)`
- `apps/web/app/api/finance/route.ts` — то же + добавлен `allPayrollOrders` для P&L по месяцам

**Расчёт долга по ЗП:**  
`debt = MAX(earned - paid, 0)`, где earned = из trip_orders, paid = из PAYROLL-транзакций.  
Если `auto_settle = true` → `debt = 0` всегда.

**SQL-скрипт первичного расчёта:**  
`supabase/seed/04-settle-payroll-debts.sql` — вставляет PAYROLL-транзакции для всех текущих долгов.  
Уже применён в БД 09.05.2026.

## 5. Данные нужные от пользователя

### 5.1 Кредиты — параметры для каждого кредита:

```
1. Кредитор           — название банка или организации (пр: «Сбербанк», «ВТБ Лизинг»)
2. Тип                — кредит / лизинг / займ
3. Назначение         — на что брался (пр: «Валдай 2022», «оборотные средства»)
4. Исходная сумма     — сколько взяли изначально (в рублях)
5. Остаток на сегодня — сколько ещё осталось выплатить
6. Ставка             — процент годовых (пр: 14.5%)
7. Ежемесячный платёж — фиксированная сумма в месяц
8. Дата начала        — когда взяли (пр: 15.03.2023)
9. Дата окончания     — когда закрывается (пр: 15.03.2028)
10. Примечание        — любая доп. информация (необязательно)
```

### 5.2 Постоянные расходы машин (`monthly_fixed_cost`):

Заполняется через UI `/fleet` → кнопка «Изменить» на каждой машине.  
Поле = ОСАГО/12 + транспортный налог/12 + платёж по лизингу/кредиту.

## 6. Ключевые технические факты

- `users.roles` — массив, фильтровать через `.contains('roles', ['driver'])`, НЕ `.eq('role', ...)`
- Все API-роуты используют `createAdminClient()` с `(supabase.from('table') as any)`
- Деньги — только строкой: `"5000.00"`, компонент `<Money amount="5000.00" />`
- `crypto.randomUUID()` для idempotency_key в webapp (НЕ пакет `uuid`)
- CSS Grid для плиток: всегда `items-start` — иначе все карточки строки растягиваются
- `shrink-0` на метриках + `flex-1 min-w-0` на идентификаторе — выравнивание колонок между карточками
- Двухшаговое удаление: `boolean state` + `useRef` таймер 4 сек авто-сброс
- `useSearchParams()` в Next.js 15 обязательно в компоненте обёрнутом в `<Suspense>` — иначе падает сборка
- Генерация TypeScript-типов: `npx supabase gen types typescript --project-id dzwuvbhfqnsxbfbynmgz > packages/shared-types/src/database.types.ts`
- После генерации типов — проверить что в конец файла не вставилась строка CLI-предупреждения (ломает TS)
- Деплой: Vercel автодеплой из `main` для обоих приложений
