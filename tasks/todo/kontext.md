## 1. Что за проект

SaldaCargo — ERP-система для транспортной компании (Верхняя Салда).  
Цель: в будущем продать как коробочное решение для других транспортных компаний.

**Два приложения:**

- `apps/miniapp` — Next.js 15, мобильный интерфейс для водителей/механиков/админа (мессенджер MAX)
- `apps/web` — Next.js 15, десктопный дашборд для владельца и админа

**Стек:** Next.js 15, React 19, TypeScript, Supabase (PostgreSQL), TanStack Query, Tailwind CSS 4  
**Auth:**

- MiniApp: кастомная cookie `salda_user_id` (UUID из `users`)
- WebApp: кастомная cookie `salda_auth_token` (UUID из `users`) — через `/login` (телефон + PIN)

**Деплой:** Vercel (miniapp и webapp — оба задеплоены, автодеплой из `main`)

## 3. Гараж WebApp — текущее состояние

### Реализовано (сессии 15–16)

- **Жизненный цикл нарядов:**
  - `lifecycle_status: draft` — наряд открыт
  - `lifecycle_status: approved` — наряд закрыт (Архив)
  - `lifecycle_status: cancelled` — удалён (пароль 9111, soft-delete)
- **Виды работ:**
  - Справочник `work_catalog` (95 позиций, 10 категорий, поиск)
  - Поле `quantity` (кол-во единиц): цена = норм.мин × qty / 60 × нормачас
  - Цена только автоматическая (ручной ввод убран)
- **Закрытие работы:**
  - «✓ Выполнено» только если механик назначен (проверка `order.mechanic || editMechanic`)
  - Транзакция PAYROLL_MECHANIC `settlement: pending`: `price_client × mechanic_salary_pct / 100`
  - 2 механика → каждый 50% × свой процент
- **Явное закрытие наряда:**
  - Авто-закрытие УБРАНО
  - Кнопка «Завершить наряд» появляется когда все работы `completed`
  - После нажатия → `lifecycle: approved` → Архив
- **Архив:** вкладка + «Открыть заново» (→ draft) + удаление (пароль 9111)
- **Печать A4:** `/api/garage/orders/[id]/print`

## 5. Архитектура учёта нарядов (НОВАЯ, сессия 15)

### Жизненный цикл наряда

```
Создание → lifecycle_status: draft, status: created
Работы добавлены → работы в статусе pending
Механик/Админ выполняет работу → status: completed, salary_paid: true
  → AUTO: транзакция PAYROLL_MECHANIC (settlement: pending = долг)
Все работы completed → Кнопка «Завершить наряд» (явная!)
  → lifecycle_status: approved → Архив
Открыть заново → lifecycle_status: draft
Удалить (пароль 9111) → lifecycle_status: cancelled
```

### Расчёт ЗП за работу

```
workPrice = price_client (авто: факт.мин × нормачас × qty)
salary = workPrice × mechanic_salary_pct / 100
Если 2 механика: каждый = (workPrice / 2) × pct / 100
settlement_status: 'pending' = долг (НЕ выплата)
```

### Нормачас

- Клиентский автомобиль: `sto_settings.hourly_rate` (2000 ₽/ч)
- Собственный автомобиль: `sto_settings.hourly_rate_own` (1600 ₽/ч)

## 8. Архитектура расчётов

| Раздел                | Источник выручки                                                                     | Источник расходов                                             |
| --------------------- | ------------------------------------------------------------------------------------ | ------------------------------------------------------------- |
| **P&L (/finance)**    | `trip_orders` (lifecycle=approved + settlement=completed + trips.lifecycle=approved) | `transactions` (expense, approved, completed)                 |
| **Контрагенты**       | то же (унифицировано с Finance)                                                      | —                                                             |
| **Автопарк (/fleet)** | `trip_orders.amount` (approved)                                                      | `trip_expenses` + `driver_pay/loader_pay` + `part_movements`  |
| **Балансы кошельков** | `transactions` (income, approved, completed, to_wallet_id)                           | `transactions` (expense, approved, completed, from_wallet_id) |
| **Дебиторка**         | `trip_orders` (settlement=pending) + `manual_receivables` (settled=false)            | —                                                             |

### Долги поставщиков

Долг = SUM(expense, pending, approved) − SUM(expense, completed, approved) по `counterparty_id`

```
20000000-0000-0000-0000-000000000001 — Дерябин ГСМ
20000000-0000-0000-0000-000000000002 — Новиков А.В. Запчасти
20000000-0000-0000-0000-000000000003 — Ромашин Запчасти
```

### Модель наличных

**При апруве рейса** → автоматическая income-транзакция в Сейф (TRIP_REVENUE).  
Понятия «подотчёт водителя» и «инкассация» больше не существуют.

## 9. Ключевые технические факты

- `users.roles` — массив: `.contains('roles', ['driver'])`, НЕ `.eq('role', ...)`
- Все API-роуты: `createAdminClient()` + `(supabase.from('table') as any)`
- Деньги — только строкой: `"5000.00"`, компонент `<Money amount="5000.00" />`
- `crypto.randomUUID()` для idempotency_key
- **WebApp auth:** `salda_auth_token` cookie (НЕ `salda_user_id`!) — читать через `requireAuth()` из `lib/auth.ts`
- **NULL в PostgreSQL:** `.neq('col', 'x')` не находит строки с `NULL`. Использовать `.or('col.is.null,col.neq.x')`
- Скроллируемый flex-child: обязательно `min-h-0` + `overflow-y-auto`

### ID кошельков
