Ты — Senior-разработчик. Твоя задача — написать модуль для автоматической генерации PDF-документа (Заказ-наряда для автосервиса) на основе входящих данных в формате JSON.

### 1. ТЕХНИЧЕСКИЙ СТЕК:

Моя ERP-система написана на: [УКАЖИ ЯЗЫК, например: PHP 8 / Node.js / Python]
Используй самую надежную и популярную библиотеку для этого языка (например, mPDF/TCPDF для PHP, Puppeteer/PDFKit для Node.js, ReportLab/WeasyPrint для Python).
Скрипт должен принимать JSON, парсить его и отдавать готовый файл .pdf или ссылку на него.

### 3. ТРЕБОВАНИЯ К ВЕРСТКЕ И ДИЗАЙНУ PDF:

Формат: А4, книжная ориентация, строгий черно-белый стиль для печати.
Шрифт: Обязательно с поддержкой кириллицы (например, DejaVu Sans, Arial или Roboto).

Документ должен состоять из следующих блоков сверху вниз:

1. ШАПКА:

- Крупно по центру: "ЗАКАЗ-НАРЯД № [Сгенерируй случайный номер или таймстемп] от [Текущая дата]"
- Ниже две колонки. Левая: Исполнитель (Автосервис "КомТранс-Салда", г. Верхняя Салда). Правая: Данные клиента (Марка авто, Модель, Госномер, Пробег).

2. ТАБЛИЦА 1: ВЫПОЛНЕННЫЕ РАБОТЫ

- Колонки: № | Наименование работы (сгруппировано по полю `group`) | Нормочасы (переведи `norm_minutes` в часы, т.е. раздели на 60) | Цена | Сумма
- Под названием работы мелким и курсивным шрифтом выводи `work_description`.
- Под таблицей: "Итого по работам: [totals.works] руб."

3. ТАБЛИЦА 2: ЗАПАСНЫЕ ЧАСТИ И МАТЕРИАЛЫ

- Колонки: № | Наименование | Кол-во | Ед. изм. | Цена | Сумма
- Под таблицей: "Итого по запчастям: [totals.parts] руб."

4. СВОДНЫЕ ДАННЫЕ (Выравнивание по правому краю):

- Слесарные усложнения: [complications] руб.
- ИТОГО К ОПЛАТЕ: [totals.total] руб. (Выделить жирным, крупнее остального текста).

5. ПОДВАЛ (RECOMMENDATIONS & WARRANTY):

- Блок "Рекомендации мастера": вывести списком из массива `recommendations`.
- Если есть `delivery_note`, вывести красным или жирным: Внимание по доставке.
- Гарантийные обязательства: вывести текст из `warranty_note`.

6. ПОДПИСИ СТОРОН:

- В самом низу две линии для подписей: "Мастер-приемщик ****\_****" и "Заказчик (с объемом работ согласен) ****\_****"

### 4. ОСОБЫЕ УСЛОВИЯ ДЛЯ КОДА:

- Код должен быть чистым, с комментариями.
- Обеспечь защиту от "разрыва" строк в таблицах при переносе на новую страницу (page break).
- Напиши не только функцию генерации PDF, но и пример её вызова с моим JSON.
