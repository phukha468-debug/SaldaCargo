# 🛠 ЗАДАЧА 34: Устранение ложных разлогинов, фикс ошибки JSON и чистка двойного выбора машины

## 1. Контекст и Проблема
**Проблема 1 (SyntaxError):** При выборе роли "Водитель" на стартовом экране происходит попытка загрузить машины через `fetch('/api/driver/assets')`. Сервер возвращает HTML (вероятно, из-за блокировки RLS или редиректа Middleware на API), что вызывает ошибку `Unexpected token '<'`.
**Проблема 2 (Ложный разлогин):** Страница `apps/miniapp/app/driver/page.tsx` принудительно разлогинивает пользователя. Она запрашивает `/api/driver/me`, ловит `401` (т.к. мы используем упрощенный вход через куку без Supabase Auth Token) и удаляет `salda_user_id`.
**Проблема 3 (Двойной выбор машин):** Выбор машин должен остаться **ТОЛЬКО** на стартовом экране.

## 2. Пошаговый план исправления (Выполнить строго по порядку)

### Шаг 1: Фикс ошибки "Unexpected token '<'" при загрузке машин (Стартовый экран)
**Файл:** `apps/miniapp/app/page.tsx`
- Сейчас `fetch` стучится в закрытый API. Чтобы это обойти, мы загрузим машины через публичный API.
- Создай новый публичный роут: `apps/miniapp/app/api/vehicles/public/route.ts`.
- Внутри этого роута используй `createAdminClient` (как в `users/public`), чтобы обойти RLS и вытащить список всех активных машин из таблицы `vehicles` (поля `id`, `name`, `number`).
- Обнови запрос в `app/page.tsx`: замени `fetch('/api/driver/assets')` на `fetch('/api/vehicles/public')`.

### Шаг 2: Отключение ложного разлогина в профиле водителя
**Файл:** `apps/miniapp/app/driver/page.tsx`
- Найди запрос `useQuery` с ключом `['me']` к `/api/driver/me`.
- **Полностью закомментируй или удали** блок `if (res.status === 401) { ... }`, который удаляет куку `salda_user_id` и выбрасывает `Error('Unauthorized')`. На этапе MVP этот запрос не должен «убивать» сессию. 
- *Примечание: Если запрос к `/api/driver/me` падает из-за RLS, в будущем мы переведем его на использование куки вместо Supabase Auth, но сейчас главное — остановить удаление куки.*

### Шаг 3: Удаление старого выбора машин из интерфейса водителя
**Файл:** Компоненты внутри `apps/miniapp/app/driver/...` (скорее всего, в `layout.tsx` или в самом `page.tsx`).
- Найди UI-компонент, который повторно запрашивает у водителя выбор автомобиля перед началом рейса.
- **Полностью удали** этот компонент/модальное окно.
- Логика должна быть такой: водитель выбрал машину на стартовом экране, мы сохранили её `id` (например, в `localStorage`), и внутри интерфейса `/driver` мы просто читаем это значение. В интерфейсе водителя можно оставить только кнопку "Выбрать грузчиков" (если она там есть).

## 3. Ожидаемый результат
1. При выборе роли "Водитель" список машин грузится без ошибок `SyntaxError`.
2. После входа в `/driver` пользователя **НЕ** выбрасывает обратно на страницу ввода ПИН-кода.
3. Внутри раздела `/driver` больше не появляется окно выбора автомобиля.
