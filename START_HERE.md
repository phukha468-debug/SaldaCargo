# START HERE — Запуск проекта SaldaCargo

**Это твоя персональная инструкция.** AI-ассистенты её НЕ читают — они работают по `TASK_*.md` файлам.

Ты делаешь только три вещи:

1. **Кладёшь файлы в `C:\salda`** (один раз, в начале).
2. **Передаёшь задания Claude Code** по очереди из папки `tasks/todo` (сначала `TASK_00`, потом `TASK_01`, и так далее). Выполненные задания перемещаются в `tasks/done`.
3. **Настраиваешь сервисы** (Supabase, Vercel, МАХ) когда задание это потребует.

## Шаг 3. Передавать задания Claude Code

### Как запустить Claude Code

Открой PowerShell, перейди в папку проекта:

```powershell
cd C:\salda
claude
```

При первом запуске пройдёшь авторизацию через Anthropic.

### Как передавать задание

В Claude Code пишешь **точно такую фразу** (заменяя номер задания):

```
Прочитай файл tasks/todo/TASK_00_BOOTSTRAP_AND_DISTRIBUTE.md и выполни его пошагово.
В конце покажи отчёт: что сделал, что не получилось, какие есть вопросы.
Не делай ничего, что не указано в задании.
```

Claude Code прочитает задание, выполнит его и покажет отчёт. Ты:

1. **Проверяешь результат** (по критериям приёмки в задании).
2. **Если всё ок** — перемещаешь файл задания в `tasks/done` и переходишь к следующему заданию.
3. **Если есть проблемы** — даёшь Claude Code уточнения, либо пишешь мне сюда, в чат.

## Когда что-то идёт не так

### Сценарий 1: Claude Code не понял задание

Скажи ему:

```
Прочитай ещё раз файл TASK_XX внимательно. Особенно секцию "Критерии приёмки".
Потом расскажи, что именно ты будешь делать на каждом шаге, и спроси у меня
подтверждения, прежде чем что-то менять в файлах.
```

Если он всё равно делает не то — пиши мне в чат, я переработаю задание.

### Сценарий 2: Возникла ошибка `pnpm install` или `pnpm build`

Скажи Claude Code:

```
Запусти pnpm install и pnpm build. Если что-то падает — покажи мне полный вывод
ошибки и предложи решение. Не пытайся "обойти" ошибку, если не уверен в причине.
```

### Сценарий 3: Что-то сломалось после задания

Откати последний коммит:

```powershell
git reset --hard HEAD~1
```

И попробуй задание заново. Если повторяется — пиши мне.

### Сценарий 4: Claude Code достиг лимита токенов в сессии

Это нормально на больших заданиях. Закрой Claude Code, открой заново, скажи:

```
Я продолжаю работу над проектом. Прочитай README.md и CLAUDE.md, чтобы войти
в контекст. Я работаю над заданием TASK_XX. Вот что уже сделано: [опиши кратко].
Продолжи с этого места.
```

## Главные правила работы с AI-ассистентом

1. **Одно задание за раз.** Не запускай TASK_02 пока TASK_01 не выполнен.
2. **Проверяй критерии приёмки.** В каждом TASK есть секция «Критерии приёмки» — пройдись по ним.
3. **Коммить после каждого задания.** В конце каждого TASK есть инструкция коммита.
4. **Если AI пишет код, который не упомянут в задании — стопаешь его.** «Делай только то, что написано в TASK_XX».
5. **Не торопись.** Лучше 2-3 часа в день и без ошибок, чем 8 часов и потом неделя на исправления.

## Что хранить в голове

- Все источники правды по бизнесу — в `docs/business/`. Не правь данные напрямую в БД.
- ЗП — ручной ввод, формулы только подсказывают.
- Транзакции — двухосный статус (lifecycle + settlement). Балансы считаются только по `approved + completed`.
- Деньги — `string`/`DECIMAL`, никогда `number`/`float`.
- Soft-delete: статус `cancelled`, не `DELETE`.
- Ретроспективные данные за 19 апреля — 2 мая ты будешь вводить **сам через WebApp** в Шаге 15.
